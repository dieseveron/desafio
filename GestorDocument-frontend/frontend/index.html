Gestor de Documentos

Se desea diseñar e implementar una solución Web para el módulo administración de documentos como parte del gestor de documentos corporativo. A continuación se detallan todos los requerimientos funcionales que deberá cumplir este módulo: 1. Alta de documentos: Para el alta de documentos se presentará un formulario con los siguientes campos: File input que permita seleccionar documentos alojados en la máquina del operador. Combo que permita seleccionar el tipo de documento. Los tipos de documentos son: Público, Privado, Draft. Combo que permita seleccionar el owner del documento. Campo que permita ingresar la descripción del documento. Todos los campos son requeridos excepto descripción. 2. Listado de documentos: 2.a) Los documentos se deberán presentar en forma de listado con la siguiente estructura: Nombre del documento Fecha de subida Tipo de documento Owner del documento Documento (Descargable) 2.b) poder consultar de forma externa tipo Api Rest el servicio de listado de documentos (Utilizar Spring Rest) 3. Editar documentos: Ídem alta. 4. Eliminar documentos: Se deberá poder borrar de forma consistente los documentos cargados. 5. Detalle del documento: 5.a) El detalle de cada documento se debe mostrar en un diálogo con los siguientes datos: Nombre (el nombre debe ser un link que permita hacer download del documento) Fecha de subida Owner Descripción Grupo de usuarios relacionados con el documento 5.b) poder consultar de forma externa tipo Api Rest el servicio de listado de documentos (Utilizar Spring Rest) Condiciones generales: Se debe resolver utilizando Java, JPA, Spring, En caso de no conocer alguna de las tecnologías, usar la que considere. En los puntos 2.b) y 5.b) será necesario devolver un Json Te pedimos que una vez finalizado nos compartas por github o donde consideres.
